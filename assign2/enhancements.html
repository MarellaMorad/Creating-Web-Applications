Enhancements

Usability, Functionality and Maintainability Enhancements

These enhancements aim to improve the usability, functionality and maintainability of this website. Various features such as validation indicators, new HTML elements, and color palettes have been added to enhance the user experience. These improvements were made to create a more engaging and interactive experience for users.

Validation Indicators and Style-changing Navbar using CSS

These changes have been implemented to provide the user with additional feedback and interactivity, as well as enhancing the visual design of the website.

Validation indicators have been added to form inputs including:

  • The red * for required fields implemented using the CSS ::after pseudo-element - see jobs.html.
  • Having input fields highlighted in red with a red border when the input is invalid. This is implemented using the CSS :invalid pseudo-class - see jobs.html.

Also, I made some style changes for the navbar to indicate for example:

  • The current page - background color of the navbar item changes to purple and text color changes to white which is implemented using a custom css class .current that indicates the current page - try selecting a different page and see how the styling changes!
  • This change is also visible on hover which is implemented using the CSS :hover pseudo-class - try hovering over different menu item and see how the styling changes!.

Third-party sources used include the W3C HTML specification for CSS Pseudo-classes and Pseudo-elements as well as the lecture notes from weeks 3 and 4.

New HTML Elements (iframe, audio and details)

These new HTML elements are in place to improve the user experience by providing additional interactivity, multimedia content, and organization of information.

  • An iframe element has been added to embed and display a Facebook video of me singing which is part of the aboutMe.html page
  • An audio element has been added to play an voice-note of me showing my bilingual skills.
  • Multiple details elements have been added to allow content to be collapsed and expanded by the user. This enhances interactivity allowing users to select the content they want to view. I also used the open Boolean attribute indicates whether or not the details element contents are currently visible. You can see this on the Position Description page as well as this page

Third-party sources used include the W3C HTML specification for iframe, audio, and details tags.

Specify a Consistent Color Palette

This helps establish a visual identity, improve the user experience, simplify the design process and improve maintainability and consistency of the website. It is a small but impactful detail that can make a big difference in the overall look and feel of a website

It goes beyond the basic requirements of the assignment by demonstrating a commitment to consistent branding and design, which can improve the user experience and make the website more memorable.

To specify a consistent color palette for the website, I used the CSS :root pseudo-class to define variables for each color, such as light-purple, dark-purple.... Then, I used these variables throughout my CSS code to apply consistent colors to various HTML elements.

I referred to the CSS-Tricks website to learn more about using the :root pseudo-class and defining variables in CSS

You can see the implementation of this enhancement throughout my website. The colors used in the header, footer, buttons, and other elements are consistent across all web pages, providing a unified look and feel.

Responsive Design

A responsive website provides a better user experience to visitors by making the website easily accessible and readable on any device. It ensures that the content and functionality of the website are easily accessible and optimized for the screen size of the device being used. This enhances user engagement and encourages visitors to stay on the website longer.

The use of @media queries

Making the website responsive is an important feature that goes beyond the basic requirements of the assignment as it ensures that the website is easily accessible on different screen sizes, ranging from desktops to mobile devices.

To make the website responsive:

  1. I first needed to add a meta tag in my HTML to support and respond correctly to the @media queries
  2. Then I was able to use @media queries in my CSS code to control the HTML elements on certain screen sizes. I set a breakpoint for medium screens at 869px and for small screens at 450px.

After setting up the @media queries in my CSS code, I defined some HTML elements that will only show on smaller/small screens, such as the small-menu section

I referred to the Mozilla Developer Network to implement responsive design in my website and learn how to use the @media queries.

You can see the implementation of responsive design throughout my website. For example, if you resize the browser window or view the website on a mobile device, you will notice that the layout and styling of the website adjusts accordingly. One main thing that you may notice is the navbar items changing from text (on large screens) to Font Awesome icons (on medium and small screens).

Font Awesome Icons

Font Awesome has been used as an external stylesheet to provide icons for the website's menu on smaller screens, as well as links to the my social media profiles. This enhances visual interest and functionality

Code needed to implement this includes:

  1. Linking to the Font Awesome stylesheet in the HTML document's head section.
  2. Adding span elements and setting their classes according to the Font Awesome documentations.

You can see this enhancement in action on the navbar menu (available on all web pages on medium and small screens) and the social media links on the aboutMe page - right above the footer.
